# CS-370
Pirate Intelligent Agent Project Reflection

Briefly explain the work that you did on this project: What code were you given? What code did you create yourself?
For this project, I worked on creating an intelligent agent that could find treasure while avoiding pirates in a grid style environment. I was given a starter Jupyter Notebook that already had the environment set up, including the grid, rewards, and some helper functions. My job was to build the actual learning part of the agent. I implemented the deep Q-learning model, set up the neural network, and created the training loop so the agent could learn from its actions. I also adjusted things like exploration rate and learning rate to improve how well the agent performed over time.

What do computer scientists do and why does it matter?
Computer scientists basically solve problems using technology. They design systems, write algorithms, and build software that can handle complex tasks. This matters because a lot of things we use every day ike apps, AI tools, and even navigation systems—depend on these kinds of solutions to work efficiently and reliably.

How do I approach a problem as a computer scientist?
I usually start by trying to fully understand the problem, then break it down into smaller pieces. After that, I think about what tools or algorithms I can use to solve each part. This project showed me that trial and error is a big part of the process, especially with machine learning. You don’t always get the best result right away, so testing and adjusting things is really important.

What are my ethical responsibilities to the end user and the organization?
I think it’s important to make sure whatever I build is safe, reliable, and fair for users. That means testing things properly and being aware of how the system might behave in different situations. There’s also a responsibility to avoid bias and make sure the system doesn’t cause harm. Overall, it’s about building something that people can trust and that works the way it’s supposed to.
